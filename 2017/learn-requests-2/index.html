高级用法

本次的高级用法涵盖了一些 Requests 的高级特性。

会话对象

会话对象让你能够跨请求保持某些参数。它也会在同一个 Session 实例发出所有请求之间保持 cookie，期间使用了 urllib3 的 connection.pooling 功能。所以如果你向同一个主机发送多个请求的时候，底层的 TCP 连接将会被重用，从而带来显著的性能提升。

请求与响应对象

任何时候进行类似于 requests.get() 的调用，你都可以做两件事情。其一，你在构建一个 Requests 对象，该对象将被发送到某个服务器请求或查询一些资源。其二，一旦 requests 得到一个从服务器返回的响应就会产生一个 Response 对象。该响应对象包含服务器的所有信息，也包含你原来创建的 Requests 对象。如下是一个简单的请求，从 Wikipedia 的服务器得到一些非常重要的信息：

你可以进一步使用 Response.iter_content 和 Response.iter_lines 方法来控制工作流，或者以 Response.raw 从底层urllib3 的 urllib3.HTTPResponse <urllib3.response.HTTPResponse 读取未解码的相应体。

如果你在请求中把 stream 设为 True，Requests 无法将连接释放回连接池，除非你 消耗了所有的数据，或者调用了 Response.close。 这样会带来连接效率低下的问题。如果你发现你在使用 stream=True 的同时还在部分读取请求的 body（或者完全没有读取 body），那么你就应该考虑使用 with 语句发送请求，这样可以保证请求一定会被关闭：

小结

高级用法的功能比较复杂，还比较多，理解起来还是比较费力的，所以还是分了两次再写吧。这也是面向网络编程的一部分，是必不可少的。尤其是会话对象这个用起来是非常方便的，可以保持会话的连接，不用管其他的东西。
